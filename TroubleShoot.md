# TroubleShoot.md

본 문서는 프로젝트 개발 및 운영 중 발생한 기술적 문제와 그 해결책을 기록합니다.

## 1. FRED API 데이터 누락 이슈
- **증상**: 일부 공휴일에 데이터 수집 시 값이 `.`으로 반환되어 파싱 에러 발생.
- **해결**: `fred_client.go`에서 `.` 값을 체크하여 스킵하는 로직 추가.

## 2. 점수 산정 로직 불일치 (2026-04-11)
- **증상**: `Score Standard.md` 문서의 11개 지표 기준과 `main.go`의 9개 지표 코드가 상이함.
- **원인**: 초기 개발 시 지표 누락 및 가중치 계산 오류.
- **해결**: `main.go` 및 `scoring.go`를 리팩토링하여 11개 전 지표를 수용하고 문서 가중치와 동기화함.

## 3. 환경 변수 로드 경로 문제
- **증상**: 서버 실행 시 `.env` 파일을 찾지 못해 API 키 로드 실패.
- **원인**: 서버 실행 위치에 따라 상대 경로가 달라짐.
- **해결**: `godotenv.Load("../.env")`를 통해 루트 디렉토리의 파일을 명시적으로 로드하도록 수정.

## 4. npm run dev 실행 시 ENOENT 에러
- **증상**: `Could not read package.json: Error: ENOENT: no such file or directory` 에러 발생.
- **원인**: 프로젝트 루트(`USA Graph`)에서 명령어를 실행함. 루트에는 `package.json`이 없고 `client/` 폴더 내에 존재함.
- **해결**: `cd client` 명령어로 이동 후 `npm run dev`를 실행하도록 가이드 보강.

## 5. 프론트엔드 차트 렌더링 부하
- **증상**: 11개 지표 카드가 동시 렌더링될 때 브라우저 프레임 드랍 발생.
- **해결**: `MiniChart` 컴포넌트의 `maintainAspectRatio: false` 설정 및 불필요한 애니메이션 옵션 조정을 통해 최적화.
